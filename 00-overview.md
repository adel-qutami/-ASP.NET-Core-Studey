# نظرة عامة على ASP.NET Core وبيئة العمل

## المقدمة

ASP.NET Core هو إطار عمل مفتوح المصدر ومتعدد المنصات (Cross-platform) من Microsoft لبناء تطبيقات ويب حديثة تعتمد على السحابة (Cloud-based) ومتصلة بالإنترنت. يوفر أداءً عالياً، مرونة في النشر، وقابلية للتوسع، مما يجعله خياراً ممتازاً لتطوير خدمات الـ Back-end، واجهات برمجة التطبيقات (APIs)، وتطبيقات الويب.

## لماذا ASP.NET Core؟

| الميزة | الوصف |
| --- | --- |
| **متعدد المنصات** | يعمل على Windows, Linux, و macOS. |
| **مفتوح المصدر** | مجتمع نشط ومساهمات مستمرة. |
| **أداء عالي** | مصمم للأداء والكفاءة، خاصة في بيئات السحابة. |
| **مرونة النشر** | يمكن نشره ذاتياً (Self-contained) أو كإطار عمل تابع (Framework-dependent). |
| **حقن التبعية (DI)** | دعم مدمج لنمط حقن التبعية، مما يسهل الاختبار والصيانة. |
| **وحدات قابلة للتكوين** | بنية معيارية تسمح بإضافة الميزات حسب الحاجة. |

## بيئة العمل الأساسية

لتطوير تطبيقات ASP.NET Core، ستحتاج إلى الأدوات التالية:

### 1. .NET SDK

مجموعة تطوير البرامج (Software Development Kit) التي تحتوي على كل ما تحتاجه لبناء وتشغيل تطبيقات .NET، بما في ذلك الـ Runtime، المكتبات، والـ CLI (Command Line Interface).

```bash
# للتحقق من تثبيت .NET SDK
dotnet --version
```

### 2. محرر أكواد (IDE)

*   **Visual Studio (Windows/macOS):** بيئة تطوير متكاملة وغنية بالميزات، مثالية للمشاريع الكبيرة. يوفر أدوات قوية للتصحيح (Debugging)، إدارة قواعد البيانات، والنشر.
*   **Visual Studio Code (Cross-platform):** محرر أكواد خفيف الوزن وقوي، مدعوم بإضافات ممتازة لتطوير .NET. مناسب للمشاريع الصغيرة والمتوسطة، وللمطورين الذين يفضلون بيئة عمل مرنة.
*   **Rider (Cross-platform):** بيئة تطوير متكاملة مدفوعة من JetBrains، معروفة بأدواتها الذكية لتحليل الكود وإعادة الهيكلة (Refactoring).

### 3. قاعدة بيانات

يمكن لـ ASP.NET Core العمل مع مجموعة واسعة من قواعد البيانات. الأكثر شيوعاً:

*   **SQL Server:** قاعدة بيانات علائقية قوية من Microsoft، شائعة في بيئات الشركات.
*   **PostgreSQL:** قاعدة بيانات علائقية مفتوحة المصدر، معروفة بموثوقيتها وميزاتها المتقدمة.
*   **MySQL:** قاعدة بيانات علائقية مفتوحة المصدر، سهلة الاستخدام وشائعة لتطبيقات الويب.
*   **SQLite:** قاعدة بيانات خفيفة الوزن ومدمجة، مثالية للتطبيقات الصغيرة أو التطوير المحلي.

## إنشاء أول مشروع ASP.NET Core Web API

لنبدأ بإنشاء مشروع Web API جديد باستخدام .NET CLI:

```bash
# إنشاء مجلد للمشروع
mkdir MyFirstApi
cd MyFirstApi

# إنشاء مشروع Web API جديد
dotnet new webapi -n MyFirstApiProject

# استعادة التبعيات (Dependencies)
dotnet restore

# تشغيل المشروع
dotnet run
```

بعد تشغيل المشروع، ستتمكن من الوصول إلى API الخاص بك عادةً على `https://localhost:5001` (أو منفذ آخر يحدده النظام). يمكنك اختبار نقطة النهاية الافتراضية `WeatherForecast`.

## هيكلية المشروع الأساسية

عند إنشاء مشروع Web API جديد، ستحصل على الهيكلية التالية:

```
MyFirstApiProject/
├── Controllers/             # المتحكمات التي تعالج طلبات HTTP
│   └── WeatherForecastController.cs
├── Properties/              # إعدادات المشروع (مثل launchSettings.json)
│   └── launchSettings.json
├── appsettings.json         # ملفات إعدادات التطبيق
├── appsettings.Development.json
├── Program.cs               # نقطة الدخول الرئيسية للتطبيق
├── MyFirstApiProject.csproj # ملف المشروع (يحتوي على التبعيات وإعدادات البناء)
└── WeatherForecast.cs       # نموذج بيانات افتراضي
```

*   **`Program.cs`:** هو الملف الذي يتم فيه تهيئة التطبيق، وتكوين الخدمات (Services)، وبناء الـ Request Pipeline.
*   **`Controllers`:** يحتوي على الفئات التي تتعامل مع طلبات HTTP الواردة وتُرجع الاستجابات. كل دالة (Action) في المتحكم تقابل نقطة نهاية (Endpoint) في الـ API.
*   **`appsettings.json`:** يستخدم لتخزين إعدادات التطبيق، مثل سلاسل الاتصال بقواعد البيانات أو مفاتيح الـ API.

## الخلاصة

ASP.NET Core يوفر بيئة قوية ومرنة لتطوير تطبيقات الـ Back-end. فهم هذه الأساسيات هو الخطوة الأولى نحو بناء أنظمة قوية وقابلة للتوسع. في الدروس القادمة، سنتعمق في كل من هذه المكونات ونستكشف كيفية بناء API احترافي خطوة بخطوة.
